# Replication Package

**"Detecting Adoption from Output: Why First-Detection Event Studies Can Mislead — A Comment on Kusumegi et al. (2025)"**

---

## Overview

This package replicates all figures in the comment paper. The paper argues that the treatment-timing rule in Kusumegi et al. (2025) — dating LLM adoption as the first month in which at least one abstract exceeds an LLM-detection threshold — mechanically inflates post-treatment event-study coefficients. We demonstrate this via simulation (Figure 2) and placebo exercises on the original arXiv data (Figure 3).

---

## Data

### 1. arXiv metadata (`arxiv-metadata-oai-snapshot.json`)
Download from Kaggle:  
[https://www.kaggle.com/datasets/Cornell-University/arxiv](https://www.kaggle.com/datasets/Cornell-University/arxiv)

Place the file `arxiv-metadata-oai-snapshot.json` in this directory. It is excluded from the repository due to its size (~4 GB).

### 2. Kusumegi et al. panel data (`production_arxiv.csv`)
Download from the original paper's replication repository on figshare:  
[https://doi.org/10.6084/m9.figshare.30359437](https://doi.org/10.6084/m9.figshare.30359437)

Place `production_arxiv.csv` in this directory.

### 3. Synthetic LLM abstracts (`synthetic_llm.csv`)
Provided directly in this repository. Generated by `Synthetic_LLM.py` using the OpenAI Batch API (GPT-3.5-turbo): 2,000 non-AI-field arXiv abstracts per month (January–November 2022) were rewritten by the model to train the LLM detection classifier.

---

## Code

### `Stacked_Dataset.py`
Builds all stacked DiD datasets used in Figures 1 and 3 from the raw arXiv metadata:

- **LLM detection**: estimates a log-likelihood ratio classifier trained on `synthetic_llm.csv` to compute an LLM score (`alpha`) for each abstract. Authors are treated in the first month where at least one abstract exceeds the detection threshold (α > 0.1).
- **Stacked DiD construction**: assigns each author to exactly one cohort; never-treated authors receive a pseudo-cohort.
- **Output files** :
  - `stacked_replication.csv` — main replication of Kusumegi et al.
  - `stacked_placebo_random{10,20,30}.csv` — random flag placebos (detection prob. 10%, 20%, 30%)
  - `stacked_placebo_keyword{data,paper,find}.csv` — neutral keyword placebos ("data", "paper", "find")
  - `stacked_placebo_prechatgpt.csv` — pre-ChatGPT window placebo (before December 2022)


### `Synthetic_LLM.py`
Generates `synthetic_llm.csv` via the OpenAI Batch API. Users can alternatively use `synthetic_llm.csv` provided directly in this repository, which was generated using this script.

### `Fig1.do`
**Figure 1** — Stacked event study on the Kusumegi et al. data (`production_arxiv.csv`).  
Runs a PPML regressions with author and time fixed effects:
- `Fig1_Stacked.pdf`: stacked DiD (author + cohort×month + relative-time FEs)


### `Fig2.do`
**Figure 2** — Monte Carlo simulation with i.i.d. productivity and no causal effect.  
Compares first-detection timing (spurious positive path) vs. random timing (flat path) under stacked PPML.
Parameters: 100,000 simulated authors, 1,000 replications.
- `Fig2_Simulation_Stacked_100K_1000.pdf`: Monte Carlo average event-study coefficients under first-detection timing (spurious positive path) vs. random timing (flat path), with 95% confidence bands across replications


### `Fig3.do`
**Figure 3** — Placebo exercises on arXiv data (reads CSVs produced by `Stacked_Dataset.py`):
- `Fig3_Replication.pdf`: replication of the main Kusumegi et al. result
- `Fig3_Placebo_Keywords.pdf`: neutral keyword placebos (data / paper / find)
- `Fig3_Placebo_Random.pdf`: random flag placebos (p = 0.1, 0.2, 0.3)
- `Fig3_Placebo_Pre_Period.pdf`: pre-ChatGPT window placebo

---

## Replication Steps

1. Download `arxiv-metadata-oai-snapshot.json` from Kaggle and place it in this directory.
2. Download `production_arxiv.csv` from figshare and place it in this directory.
3. Run `Stacked_Dataset.py` to generate the stacked CSVs.
4. Open Stata and run `Fig1.do`, `Fig2.do`, `Fig3.do`.
